How does JavaScript work?

  • JS is a single threaded programming language.
  • It can run only one command/task at once.
  • Generally, JS works synchronized and chronological way i.e., line by line.
  • So, JS creates a call stack which create a que of task/command and run one by one.
  • But JS also has some functions that can run asynchronized way by using memory hipe and push the task in the call stack when certain condition fulfilled.

What does JavaScript eventloop do?

  • Eventloop manage the que of event/command.
  • It create a call stack to and callback que.
  • When the call stack become empty, it push the callback que to call stack.
  • And manage the memory hype to store memories.
  • Eventloop manage the overall web api data and asynchronous functions to integrate each other.

What are the difference between local storage and session storage?

  • Local storage store data for the specific domain where session storage store data for the current session.
  • Local storage data stores do not removed after close the broweser tab but the session storage data will remove once the tab close.
  • Opening a new tab will create a new session data where the local storage look for any existing storage data.
  • If we open multiple tab for the same tab, it create only single local storage where creates session storage for each tab.
  • When we close the tab, session storage will removed but the local storage not removed.
